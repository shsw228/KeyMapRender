# AGENTS.md

このファイルは KeyMapRender リポジトリで作業するエージェント向けのプロジェクト運用ルールです。

## 目的
- 変更の意図と影響範囲を明確にする
- 小さく安全な差分で進める
- ドキュメントと実装の整合性を保つ

## スコープ
- 本ファイルのルールはこのリポジトリ配下に適用する
- より上位の指示と矛盾する場合は、上位指示を優先する

## 作業開始時チェック
- まず `README.md` と `docs/specification.md` を確認する
- 既存ファイルの命名・実装スタイルに合わせる
- 影響が広い変更は段階分けして提案する

## 実装ルール
- 不要なリファクタは避け、依頼範囲に集中する
- 破壊的変更は事前に確認する
- 未コミット変更は勝手に巻き戻さない
- コメントは必要最小限にし、意図が伝わる内容だけ残す

## ドキュメント運用
- 仕様変更時は `docs/specification.md` を更新する
- 概要や手順の変更時は `README.md` を更新する
- 未確定事項は「未記載」ではなく TODO として明示してもよい

## テスト/検証
- 可能なら最小限の確認手順を実行し、結果を共有する
- 実行できない検証は理由を明記する

## コミット/PR
- コミットは依頼がある場合のみ実施する
- コミットメッセージは `[種類] 変更内容の説明` 形式で日本語
- PRタイトル・説明も日本語で簡潔に記載する
